  <action>
Implement in `app/mailsync-rs/src/imap/session.rs`:

1. **`role_for_name_attribute(attr: &NameAttribute) -> Option<&'static str>`** -- Maps RFC 6154 special-use attributes to role strings. Covers All, Sent, Drafts, Junk, Trash, Archive. No Inbox variant exists in imap-proto. Returns None for all other attributes (Custom, Extension, etc.).

2. **`role_for_folder_via_path(path_lower: &str) -> Option<&'static str>`** -- Name-based fallback lookup table. Case-insensitive matching against known folder name patterns:
   - "inbox" -> "inbox"
   - "sent" | "sent mail" | "sent items" | "sent messages" | "[gmail]/sent mail" -> "sent"
   - "drafts" | "draft" | "[gmail]/drafts" -> "drafts"
   - "trash" | "deleted" | "deleted items" | "deleted messages" | "[gmail]/trash" -> "trash"
   - "spam" | "junk" | "junk mail" | "junk e-mail" | "[gmail]/spam" -> "spam"
   - "archive" | "all mail" | "[gmail]/all mail" -> "all"

3. **`detect_folder_role(attrs: &[NameAttribute], path: &str) -> Option<String>`** -- Two-pass: first check attributes via `role_for_name_attribute`, if None then check path via `role_for_folder_via_path` on lowercase path.

4. **`is_gmail_sync_folder(attrs: &[NameAttribute], path: &str) -> bool`** -- For Gmail accounts, returns true if the folder should be synced as a Folder object (not a Label). Per user decision, the 6 whitelisted folders are: INBOX, [Gmail]/All Mail, [Gmail]/Trash, [Gmail]/Spam, [Gmail]/Drafts, [Gmail]/Sent Mail. Implementation:
   - Returns true if attrs contains NameAttribute::All, Junk, Trash, Drafts, or Sent
   - Also returns true if `path.eq_ignore_ascii_case("INBOX")` (INBOX has no special NameAttribute)
   - Returns false for all other folders (they become Label objects)

5. **`is_noselect(attrs: &[NameAttribute]) -> bool`** -- Returns true if NameAttribute::NoSelect is present.

Write unit tests for all behavior items. Tests use imap-proto NameAttribute enum directly (no mock IMAP server needed). Ensure tests cover all 6 Gmail whitelisted folders explicitly.

<task type="auto">
  <name>Task 2: Implement ImapSession connect, authenticate, and list_folders</name>
  <files>app/mailsync-rs/src/imap/session.rs</files>
  <action>
Implement the `ImapSession` struct in `app/mailsync-rs/src/imap/session.rs`:

1. **ImapSession struct** -- Wraps `async_imap::Session<T>` where T is the TLS stream type. Use a type alias or enum to support both direct TLS and STARTTLS connections.

```rust
pub struct ImapSession {
    session: async_imap::Session<Box<dyn AsyncReadWrite>>,
    capabilities: Vec<String>,
    is_gmail: bool,
}
```

Where `AsyncReadWrite` is a trait alias for `tokio::io::AsyncRead + tokio::io::AsyncWrite + Unpin + Send`.

2. **`connect(account: &Account) -> Result<Self, SyncError>`** -- Connects to the IMAP server:
   - Extract host, port, security from `account.extra["settings"]` (keys: `imap_host`, `imap_port`, `imap_security` where security is "SSL/TLS" or "STARTTLS")
   - For SSL/TLS (port 993): TCP connect -> TLS handshake using `tokio-rustls` with `rustls-platform-verifier` for OS trust store -> `async_imap::Session::new(tls_stream)`
   - For STARTTLS (port 143): TCP connect -> `async_imap::Client::new(tcp_stream)` -> issue STARTTLS -> upgrade to TLS
   - Wrap all operations with `tokio::time::timeout(Duration::from_secs(15), ...)` per IMPR-05
   - After connection, read capabilities and store them. Check for "X-GM-EXT-1" capability to set `is_gmail` flag
   - Map errors: io::Error -> SyncError::Connection, rustls errors -> SyncError::SslHandshakeFailed, timeout -> SyncError::Timeout

3. **`authenticate(&mut self, account: &Account, access_token: Option<&str>) -> Result<(), SyncError>`** -- Authenticates:
   - If `access_token` is Some: XOAUTH2 auth using `session.authenticate("XOAUTH2", xoauth2_authenticator)` with the Base64 SASL payload from `build_xoauth2_string(email, token)`
   - If None: password LOGIN using `session.login(email, password)` where password comes from `account.extra["settings"]["imap_password"]`
   - Wrap with 30s timeout
   - Map auth failures to SyncError::Authentication

4. **`list_folders(&mut self, account: &Account) -> Result<(Vec<Folder>, Vec<Label>), SyncError>`** -- Enumerates folders:
   - Call `session.list(Some(""), Some("*"))` with 30s timeout
   - Iterate results, skip NoSelect folders
   - For each folder: detect role via `detect_folder_role(attrs, path)`
   - If `is_gmail`: folders matching `is_gmail_sync_folder(attrs, path)` become Folder objects; all others become Label objects (GMAL-01). This ensures all 6 whitelisted folders (INBOX, All Mail, Trash, Spam, Drafts, Sent Mail) are synced.
   - If not Gmail: all selectable folders become Folder objects, no Labels
   - Set Folder.id as `account.id + ":" + folder.path` (matching C++ folder ID scheme)
   - Set Folder.account_id, path, role, local_status (empty JSON object on first enum)
   - Return (folders, labels) tuple

5. **`select_condstore(&mut self, path: &str) -> Result<async_imap::types::Mailbox, SyncError>`** -- SELECT with CONDSTORE:
   - Wrap `session.select_condstore(path)` with 30s timeout
   - Return Mailbox struct directly (caller extracts uid_validity, highest_modseq, etc.)

6. **`uid_fetch(&mut self, uid_set: &str, query: &str) -> Result<impl Stream, SyncError>`** -- UID FETCH wrapper:
   - Wrap `session.uid_fetch(uid_set, query)` with 120s timeout for stream creation
   - Return the stream directly (caller iterates with per-item timeouts)

7. **Expose `is_gmail` as a public getter.**

These methods are not unit-testable without a live IMAP server, so no unit tests for connect/auth. The role detection tests from Task 1 cover the pure logic. Integration testing is manual (documented in VALIDATION.md).
