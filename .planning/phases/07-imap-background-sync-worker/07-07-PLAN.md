<task type="auto">
  <name>Task 1: Implement run_sync_cycle_and_bodies() — folder enumeration and per-folder sync</name>
  <files>app/mailsync-rs/src/imap/sync_worker.rs</files>
  <action>
Replace the stub body of `run_sync_cycle_and_bodies()` (lines 506-516) with the full implementation. Remove the leading underscores from the parameter names (_session -> session, etc.). The function already has the correct signature. Implementation steps:

1. **Enumerate folders**: Call `session.list_folders(account)` to get `(Vec<Folder>, Vec<Label>)`. Save each Folder via `store.save(&mut folder)` (this emits persist deltas to the Electron UI). Save each Label via `store.save(&mut label)`. Sort folders with `sort_folders_by_role_priority(&mut folders)`.

2. **Per-folder sync loop**: Iterate over sorted folders. For each folder:
   a. Emit sync progress: `delta.emit_sync_progress(&account.id, &folder.path, idx as f64 / total as f64)`
   b. Call `session.select_condstore(&folder.path)` to get `Mailbox` (contains `uid_validity`, `highest_modseq`, `exists`).
   c. Load stored state: `let state = get_sync_state(&folder);`
   d. **UIDVALIDITY check**: If `needs_uidvalidity_reset(state.uidvalidity, mailbox.uid_validity.unwrap_or(0))`, call `store.unlink_messages_in_folder(&account.id, &folder.id).await?`, reset state to default with new uidvalidity, and set `uidvalidity_reset_count += 1`. Save the folder with updated state.
   e. **Strategy selection**: Call `select_sync_strategy(mailbox.highest_modseq)`.
   f. **CONDSTORE path**: Call `decide_condstore_action(server_modseq, server_uidnext, state.highestmodseq, state.uidnext)`. If `NoChange`, skip to next folder (had_changes stays false). If `Incremental { uid_set }` or `Truncated { uid_set }`, proceed to fetch.
   g. **UID-range path**: Use uid_set `"1:*"` (full range — the UID-range fallback fetches all UIDs and relies on local dedup by message stable ID).
   h. **IMAP FETCH**: Build fetch query string — for Gmail (`session.is_gmail()`): `"(UID FLAGS ENVELOPE INTERNALDATE X-GM-LABELS X-GM-MSGID X-GM-THRID)"`, for non-Gmail: `"(UID FLAGS ENVELOPE INTERNALDATE)"`. Use CONDSTORE modifier when available: append ` (CHANGEDSINCE {stored_modseq})` to the uid_set only for the CONDSTORE Incremental path. Call `session.uid_fetch(&uid_set, &query)`.
   i. **Stream iteration**: Use `tokio_stream::StreamExt` to iterate the fetch stream. For each `Fetch` item: call `process_fetched_message(&fetch, &[], &folder, account, session.is_gmail())` (pass `&[]` for attrs — Gmail thread_id falls back to stable ID per 07-04 decision). On success, `store.save(&mut message).await?`. If `Option<Thread>` is `Some(thread)`, check if thread exists via `store.find::<Thread>(&thread.id).await?` — if None, save the new thread. Set `had_changes = true`.
   j. **Update FolderSyncState**: After processing all fetches for a folder, update `state.uidvalidity`, `state.highestmodseq`, `state.uidnext` from the mailbox values. Call `set_sync_state(&mut folder, &state)` then `store.save(&mut folder).await?`.

3. **Body queue processing**: After the folder sync loop, if `should_cache_bodies_in_folder` is true for any synced folder, process pending body queue items. For each folder (in priority order), call `store.find_messages_needing_bodies(&account.id, &folder.id, cutoff_ts, BODY_SYNC_BATCH_SIZE as i64)` where `cutoff_ts = now - BODY_PREFETCH_AGE_SECS`. For each `(msg_id, uid)` returned, fetch the body: `session.uid_fetch(&uid.to_string(), "BODY.PEEK[]")`, parse the body bytes from the fetch result (`fetch.body()`), and update the message's `body` and `snippet` fields via `store.save()`. Also drain `body_queue.pop()` items — for each priority message ID, find its UID and folder, fetch body similarly.

4. **Emit final progress**: `delta.emit_sync_progress(&account.id, "", 1.0)` to signal cycle completion.

5. **Return**: `Ok(had_changes)`.

**Important implementation notes:**
- Use `tokio_stream::StreamExt::next()` for stream iteration (already in deps).
- The Mailbox struct from async-imap has: `uid_validity: Option<u32>`, `highest_modseq: Option<u64>`, `uid_next: Option<u32>`, `exists: u32`.
- For CONDSTORE CHANGEDSINCE, the modifier goes in the uid_fetch query, not uid_set. The correct async-imap pattern is: `session.uid_fetch("1:*", "(FLAGS) (CHANGEDSINCE 12345)")` — but this depends on async-imap's CONDSTORE support. If `uid_fetch` doesn't support modifiers natively, pass CHANGEDSINCE in the query string: `"(UID FLAGS ENVELOPE INTERNALDATE) (CHANGEDSINCE {modseq})"`. Test compilation to verify.
- Wrap individual folder sync in a match/if-let to handle per-folder errors gracefully (log warning, continue to next folder) rather than aborting the entire cycle.
- The `body_queue` parameter is `&mut BodyQueue` — drain priority items after the main folder loop.
- For body fetching, the `Fetch` struct exposes `body()` returning `Option<&[u8]>` for BODY[] data.
- Use `chrono::Utc::now().timestamp()` for the cutoff timestamp calculation.
