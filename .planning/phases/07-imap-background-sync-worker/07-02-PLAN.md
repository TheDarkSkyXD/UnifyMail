Implement `app/mailsync-rs/src/oauth2.rs` with:

1. **TokenManager struct:**
```rust
pub struct TokenManager {
    cache: HashMap<String, CachedToken>,
    http_client: reqwest::Client,
}

struct CachedToken {
    access_token: String,
    expiry_unix: i64,
}
```

2. **Core methods:**
- `TokenManager::new()` — Creates with a reqwest client using rustls
- `get_valid_token(&mut self, account: &Account, delta: &DeltaStream) -> Result<String, SyncError>` — Check cache, check 300s buffer, refresh if needed
- `refresh_token(&self, account: &Account) -> Result<TokenResponse, SyncError>` — HTTP POST to token endpoint with grant_type=refresh_token. Extract token endpoint from `account.extra["settings"]["imap_oauth_token_url"]` or fall back to provider-based defaults (Gmail: `https://oauth2.googleapis.com/token`, Microsoft: `https://login.microsoftonline.com/common/oauth2/v2.0/token`). Use `account.extra["refreshClientId"]` or env var `GMAIL_CLIENT_ID` as client_id.
- `build_xoauth2_string(username: &str, access_token: &str) -> String` — Construct XOAUTH2 SASL payload: Base64 encode `"user=<username>\x01auth=Bearer <token>\x01\x01"`

3. **Token response struct:**
```rust
struct TokenResponse {
    access_token: String,
    expires_in: Option<u64>,
    refresh_token: Option<String>,
}
```

4. **Expiry logic:** `chrono::Utc::now().timestamp() + 300 < cached.expiry_unix` means token is still valid. Otherwise refresh.

5. **Error mapping:** reqwest errors map to `SyncError::Connection` or `SyncError::Authentication` (401/403). Successful response with `error` field maps to `SyncError::InvalidCredentials`.

6. For tests, use internal helper methods that accept injected time and mock HTTP responses. Test the logic without real HTTP calls. The `refresh_request_shape` test validates the request body construction, not actual HTTP.

CRITICAL: The TokenManager must be designed for `Arc<tokio::sync::Mutex<TokenManager>>` usage (per research Pitfall 7) so Phase 8's IDLE session and background sync don't race on refresh. Document this in struct-level doc comment.

1. Add a factory method to `DeltaStreamItem` in `app/mailsync-rs/src/delta/item.rs`:

```rust
/// Creates a ProcessAccountSecretsUpdated delta for OAuth token rotation.
/// Emitted when a refresh token changes so Electron persists the new credentials.
pub fn account_secrets_updated(
    account_id: &str,
    access_token: &str,
    refresh_token: &str,
    expiry_unix: i64,
) -> Self {
    let json = serde_json::json!({
        "accountId": account_id,
        "id": account_id,
        "accessToken": access_token,
        "refreshToken": refresh_token,
        "expiry": expiry_unix,
    });
    Self::new("persist", "ProcessAccountSecretsUpdated", vec![json])
}
```

2. In `TokenManager::get_valid_token()`, after a successful refresh:
   - Compare new refresh token against `account.extra["refreshToken"]`
   - If different (rotated), call `delta.emit(DeltaStreamItem::account_secrets_updated(...))`
   - If same, do not emit
