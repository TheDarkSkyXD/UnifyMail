1. **Implement `background_sync` in `sync_worker.rs`:**

```rust
pub async fn background_sync(
    account: Arc<Account>,
    store: Arc<MailStore>,
    delta: Arc<DeltaStream>,
    token_manager: Arc<tokio::sync::Mutex<TokenManager>>,
    mut shutdown_rx: broadcast::Receiver<()>,
    mut wake_rx: mpsc::Receiver<()>,
    mut body_queue_rx: mpsc::Receiver<Vec<String>>,
) {
```

The function:
- Creates an IMAP session via `ImapSession::connect(&account)`, authenticating with password or OAuth2 token (from TokenManager via `token_manager.lock().await.get_valid_token()`)
- On auth failure (after TokenManager's internal 3-retry with exponential backoff has been exhausted): emit ProcessState with connectionError=true and wait for wake_rx or shutdown
- Maintains a `BodyQueue` for need-bodies prioritization
- Main loop:
  - Check for body_queue_rx messages (non-blocking `try_recv`), enqueue to BodyQueue
  - Run `run_sync_cycle()` to sync all folders
  - Run `process_need_bodies_queue()` if BodyQueue has items
  - Run `sync_message_bodies()` for each folder (background pre-fetch)
  - On success with changes: reset no_change_streak to 0
  - On success without changes: increment no_change_streak
  - Calculate wait: `BASE_SYNC_INTERVAL_SECS + min(no_change_streak * 30, MAX_BACKOFF_ADDITION_SECS)` seconds (~60s base, max ~300s / 5 minutes per user decision)
  - `tokio::select!` on: sleep(wait), wake_rx.recv() (reset streak), shutdown_rx.recv() (return)
  - On auth error (from get_valid_token -- already retried 3 times internally): log, emit ProcessState connectionError=true, wait for wake or shutdown
  - On retryable error: log warning, sleep 30s, continue
  - On fatal error: log error, exit process

5. **Error handling in the sync loop:**
- `SyncError::is_auth()` -> The TokenManager has already retried 3 times with exponential backoff (5s, 15s, 45s) per Plan 02. - If get_valid_token returns auth error: stop sync, emit connectionError (retries already exhausted by TokenManager)
