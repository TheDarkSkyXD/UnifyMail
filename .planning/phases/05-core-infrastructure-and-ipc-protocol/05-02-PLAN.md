    ```rust
    pub struct DeltaStream {
        tx: mpsc::UnboundedSender<DeltaStreamItem>,
    }
    impl DeltaStream {
        pub fn new(tx: mpsc::UnboundedSender<DeltaStreamItem>) -> Self { Self { tx } }
    ```

    **delta/flush.rs:** Create the dedicated flush task. This task EXCLUSIVELY owns stdout -- no other code may write to stdout.

    ```rust
    pub async fn delta_flush_task(mut rx: mpsc::UnboundedReceiver<DeltaStreamItem>)
    ```

    The flush task:
    1. Maintains a coalesce buffer: `IndexMap<String, Vec<DeltaStreamItem>>` keyed by model_class
    2. Uses `tokio::select!` between receiving items and a 500ms interval tick
    3. On item received: call `coalesce_into(&mut buffer, item)` which checks if the last item in `buffer[model_class]` has the same delta_type and can be concatenated; if yes, concatenate; if no, push new item.
    4. On interval tick (500ms): if buffer is non-empty, acquire `std::io::stdout().lock()`, write each item as a JSON line (`writeln!`), call `.flush()` on the locked stdout, then clear the buffer.
    5. On channel closed (rx.recv() returns None): flush remaining buffer and return.

    CRITICAL: Use `std::io::stdout().lock()` (synchronous), NOT `tokio::io::stdout()`. The synchronous lock+write+flush sequence is atomic and avoids tokio issue #7174 data loss. The flush task is the ONLY code that touches stdout.

    **Handshake in main.rs:** Update the main.rs Sync and Test mode dispatch to implement the two-line stdin handshake BEFORE entering the mode handler:

    ```rust
    async fn read_handshake(args: &Args) -> Result<(Account, Option<Identity>), SyncError> {
        // 1. Signal readiness (triggers TypeScript to send JSON)
        print!("\nWaiting for Account JSON:\n");
        std::io::stdout().flush()?;

        let stdin = tokio::io::stdin();
        let mut lines = BufReader::new(stdin).lines();

        // 2. Read account JSON
        let account_line = lines.next_line().await?
            .ok_or_else(|| SyncError::Protocol("stdin closed before account JSON".into()))?;
        let account: Account = serde_json::from_str(&account_line)?;

        // 3. Signal readiness for identity
        print!("\nWaiting for Identity JSON:\n");
        std::io::stdout().flush()?;

        // 4. Read identity JSON
        let identity_line = lines.next_line().await?
            .ok_or_else(|| SyncError::Protocol("stdin closed before identity JSON".into()))?;
        let identity: Option<Identity> = if identity_line == "null" {
            None
        } else {
            Some(serde_json::from_str(&identity_line)?)
        };

        Ok((account, identity))
    }
    ```
