    **1. Create `src/delta/item.rs` — DeltaStreamItem with exact serde renames:**

    ```rust
    #[derive(Debug, Clone, Serialize)]
    pub struct DeltaStreamItem {
        #[serde(rename = "type")]
        pub delta_type: String,         // "persist" or "unpersist"
        #[serde(rename = "modelClass")]
        pub model_class: String,
        #[serde(rename = "modelJSONs")]
        pub model_jsons: Vec<serde_json::Value>,
        #[serde(skip)]

    }
    ```

    CRITICAL: Use `#[serde(rename = "...")]` on EVERY field. The TypeScript parser destructures `{ type, modelJSONs, modelClass }` — any snake_case leak means silent data loss (mailsync-bridge.ts logs "unexpected keys" and skips the message).

    Implement:
    - `concatenate(&mut self, other: &DeltaStreamItem) -> bool` — merges if same delta_type and model_class, returns false otherwise
    - `upsert_model_json(&mut self, item: Value)` — finds by "id" field, merges keys (new overwrites existing, existing-only keys preserved), or appends if new id
    - `process_state(account_id: &str, connection_error: bool) -> Self` — creates ProcessState delta
    - `to_json_line(&self) -> String` — serializes to JSON string (for stdout writing)

    **2. Create `src/delta/stream.rs` — DeltaStream sender wrapper:**

    ```rust
    pub struct DeltaStream {
        tx: mpsc::UnboundedSender<DeltaStreamItem>,
    }
    impl DeltaStream {
        pub fn new(tx: mpsc::UnboundedSender<DeltaStreamItem>) -> Self;
        pub fn emit(&self, item: DeltaStreamItem);
        pub fn emit_process_state(&self, account_id: &str, connection_error: bool);
    }
    ```

    Use `tokio::sync::mpsc::unbounded_channel` — matches C++ which uses a mutex-protected vector with no capacity limit. The sender can be cloned and shared across tokio tasks via `Arc<DeltaStream>`.

    **3. Create `src/delta/flush.rs` — Dedicated flush task:**

    Implement `delta_flush_task(rx: mpsc::UnboundedReceiver<DeltaStreamItem>)`:
    - Owns stdout exclusively — NO other task writes to stdout
    - Uses `tokio::select!` between `rx.recv()` and `interval(500ms).tick()`
    - On recv: coalesce into buffer (`IndexMap<String, Vec<DeltaStreamItem>>`)
    - On tick: if buffer non-empty, flush all items to stdout then clear buffer
    - On channel close (else branch): flush remaining buffer, return
    - `flush_buffer()`: acquires `std::io::stdout().lock()`, writes each item as `writeln!(out, "{}", item.to_json_line())`, then calls `out.flush()` — CRITICAL for pipe mode
    - `coalesce_into()`: reference research Pattern 3 — check if last item in buffer[model_class] has same type, if so concatenate, otherwise push new item

    Anti-patterns to avoid (from research):
    - Do NOT mix tokio::io::stdout() with std::io::stdout() — pick one (use std::io for the flush task)
    - Do NOT write to stdout from any other task — all tracing goes to stderr
    - Do NOT use Mutex<Stdout> shared across tasks — the dedicated task pattern is correct
