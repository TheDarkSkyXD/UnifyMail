1. Create `app/mailsync-rs/src/imap/task_executor.rs`:

   - Define a trait (or use the ImapSession struct directly) for testability:
     ```rust
     /// Trait abstracting IMAP operations needed by task executor.
     /// Implemented by ImapSession for production; mock for tests.
     #[async_trait::async_trait]
     pub trait ImapTaskOps {
         async fn select_folder(&mut self, folder: &str) -> Result<(), SyncError>;
         async fn uid_store(&mut self, uid_set: &str, flags: &str) -> Result<(), SyncError>;
         async fn uid_mv(&mut self, uid_set: &str, to_folder: &str) -> Result<(), SyncError>;
         async fn uid_copy(&mut self, uid_set: &str, to_folder: &str) -> Result<(), SyncError>;
         async fn uid_expunge(&mut self, uid_set: &str) -> Result<(), SyncError>;
         async fn append(&mut self, folder: &str, flags: Option<&str>, date: Option<&str>, content: &[u8]) -> Result<(), SyncError>;
         fn has_move_capability(&self) -> bool;
         fn is_gmail(&self) -> bool;
     }
     ```
     NOTE: If async_trait is not in deps, use a concrete approach instead (pass ImapSession directly and test with integration-level stubs). The trait approach is preferred for unit testability but optional.

   - `pub async fn execute_remote_phase(session: &mut impl ImapTaskOps, task_kind: &TaskKind, account: &Account, store: &MailStore, delta: &DeltaStream, token_manager: &Mutex<TokenManager>) -> Result<(), SyncError>`:
     Match on TaskKind variants and dispatch to per-type handlers.

   - Per-type handler functions:

     a. `execute_change_starred(session, starred, uid_set, folder)`:
        - `session.select_folder(folder)`
        - If starred: `session.uid_store(uid_set, "+FLAGS (\\Flagged)")`
        - If !starred: `session.uid_store(uid_set, "-FLAGS (\\Flagged)")`

     b. `execute_change_unread(session, unread, uid_set, folder)`:
        - `session.select_folder(folder)`
        - If unread: `session.uid_store(uid_set, "-FLAGS (\\Seen)")` (remove Seen = mark unread)
        - If !unread: `session.uid_store(uid_set, "+FLAGS (\\Seen)")` (add Seen = mark read)

     c. `execute_change_folder(session, uid_set, from_folder, to_folder)`:
        - `session.select_folder(from_folder)`
        - If `session.has_move_capability()`: `session.uid_mv(uid_set, to_folder)` (RFC 6851 atomic MOVE)
        - Else fallback: `session.uid_copy(uid_set, to_folder)` + `session.uid_store(uid_set, "+FLAGS (\\Deleted)")` + `session.uid_expunge(uid_set)`

     d. `execute_change_labels(session, uid_set, folder, labels_to_add, labels_to_remove)`:
        - `session.select_folder(folder)`
        - For each label in labels_to_add: `session.uid_store(uid_set, &format!("+X-GM-LABELS ({label})"))`
        - For each label in labels_to_remove: `session.uid_store(uid_set, &format!("-X-GM-LABELS ({label})"))`

     e. `execute_destroy_draft(session, message_uid, drafts_folder)`:
        - `session.select_folder(drafts_folder)`
        - `session.uid_store(&message_uid.to_string(), "+FLAGS (\\Deleted)")`
        - `session.uid_expunge(&message_uid.to_string())`

     f. `execute_send_draft(session, account, store, delta, token_manager, task_kind)`:
        - Parse draft data via `parse_draft_data()` from task JSON
        - Build MIME email via `build_draft_email()`
        - Create SMTP sender: `SmtpSender::new(account)`
        - Get auth: check if OAuth2 via token_manager.get_valid_token() or use password
        - Build transport: `sender.build_transport(password_or_token, is_oauth2)`
        - Send: `sender.send_message(&transport, email)`
        - If NOT Gmail: APPEND to Sent folder with `(\\Seen)` flag
          `session.append("Sent", Some("(\\Seen)"), None, &SmtpSender::get_raw_message(&email))`
        - Delete the draft: mark Deleted + expunge from Drafts folder (same as DestroyDraft)
        - Emit task completion delta

     g. `execute_syncback_metadata(...)`: Return Ok(()) — metadata sync is local-only; the HTTP long-poll worker (Phase 9) handles remote sync.

     h. `execute_syncback_event(...)`: Return Ok(()) — CalDAV PUT is Phase 9 scope.

2. Wire into execute_task in tasks/mod.rs: replace the remote phase stub with a call to `execute_remote_phase()`.

3. Add `pub mod task_executor;` to imap/mod.rs.
