Purpose: Enable real-time new mail detection and task execution in the foreground, running concurrently with background sync.
Output: foreground_worker.rs with IDLE loop, updated sync.rs and stdin_loop.rs wiring.

   - Implementation (following research Pattern 1 exactly):
     a. Call crash recovery: `reset_stuck_tasks(&store).await` and log result
     b. Set up completed task expiry: spawn a tokio task that runs `expire_completed_tasks` every 60 seconds with TASK_RETENTION_SECS (900s)
     c. Connect a SEPARATE IMAP session (IDLE-03): `ImapSession::connect(&account)` then `.authenticate()`
     d. SELECT INBOX on the foreground session
     e. Enter the IDLE loop:
        ```rust
        loop {
            // Check shutdown before entering IDLE
            if shutdown_rx.try_recv().is_ok() { break; }

            let mut idle_handle = session.inner.idle();
            idle_handle.init().await?;

            // 25-minute timeout (IDLE-01)
            let (idle_future, interrupt) = idle_handle.wait_with_timeout(
                Duration::from_secs(25 * 60)
            );

            // Relay task: drops StopSource when task arrives (IDLE-02)
            let relay = tokio::spawn(async move {
                let task = task_rx.recv().await;
                drop(interrupt); // triggers ManualInterrupt
                (task_rx, task)
            });

            // Wait for IDLE to complete
            let idle_result = idle_future.await;

            // ALWAYS call done() to reclaim session (critical — research Pitfall 1)
            session.inner = idle_handle.done().await?;

            // Recover relay task
            let (recovered_rx, maybe_task) = relay.await?;
            task_rx = recovered_rx;

            match idle_result {
                Ok(IdleResponse::NewData(_)) => {
                    // New mail — signal background sync to do a quick cycle
                    let _ = wake_tx.try_send(());
                }
                Ok(IdleResponse::Timeout) => {
                    // 25 min elapsed — re-IDLE (normal operation)
                }
                Ok(IdleResponse::ManualInterrupt) => {
                    // Task arrived or channel closed — handle below
                }
                Err(e) => {
                    // IDLE error — log and attempt to reconnect
                    tracing::error!("IDLE error: {e} — reconnecting");
                    // Reconnect logic: connect + auth + select INBOX
                    // On persistent failure, sleep 30s and retry
                }
            }

            // Process any received task
            if let Some(Some((mut task, task_kind))) = maybe_task.map(Some) {
                execute_task(&mut task, &task_kind, &mut session, &store, &delta).await
                    .unwrap_or_else(|e| tracing::error!("Task execution failed: {e}"));
            }

            // Drain additional queued tasks before re-entering IDLE
            while let Ok((mut task, task_kind)) = task_rx.try_recv() {
                execute_task(&mut task, &task_kind, &mut session, &store, &delta).await
                    .unwrap_or_else(|e| tracing::error!("Task execution failed: {e}"));
            }

            // Re-SELECT INBOX before re-entering IDLE (tasks may have selected other folders)
            session.inner.select("INBOX").await?;
        }
        ```

   - Error recovery: if IDLE error or connection drop, attempt to reconnect with `ImapSession::connect` + `authenticate`. On auth error, emit connectionError ProcessState and wait for wake signal. On persistent failure (3 retries), sleep 30s then retry.

   - NOTE on `session.inner`: The ImapSession struct wraps `async_imap::Session<TlsStream<TcpStream>>`. The IDLE API requires direct access to the inner session. If ImapSession does not expose the inner session, add a `pub fn inner_mut(&mut self) -> &mut async_imap::Session<...>` accessor or restructure to allow IDLE calls. Alternatively, foreground_worker may construct its own `async_imap::Session` directly using the connect_imap pattern from ImapSession.
