<decisions>
## Implementation Decisions

### Smoke test depth
- **providerForEmail**: Functional call — call with `'test@gmail.com'` and verify a result is returned (proves loading AND data integrity). This is synchronous with embedded data, no network needed
- **registerProviders**: Type-check only (`typeof === 'function'`) — providers are already auto-loaded at import time, calling it would require a valid JSON path for no additional confidence
- **testIMAPConnection**: Type-check only — async network function, no IMAP server available in CI
- **testSMTPConnection**: Type-check only — async network function, no SMTP server available in CI
- **validateAccount**: Type-check only — async network function, no servers available in CI
- **Output style**: Verbose per-check logging — log each export check with `PASS: <export> is function` or `PASS: providerForEmail returned result`. Makes CI logs easy to scan when debugging failures

### Test script organization
- **Shared JS script file** at `app/mailcore-rs/test/smoke.js` — all 4 workflows call `node app/mailcore-rs/test/smoke.js`. One place to update when exports change. DRY and locally runnable
- **Use default shell** in CI steps — the command `node app/mailcore-rs/test/smoke.js` works identically in bash, PowerShell, and sh. No need to force a specific shell
- **Keep binary size check separate** — the 8MB size check is Linux-only and uses shell tools. It stays as its own CI step, not mixed into the cross-platform Node.js smoke script

### macOS cache key fix
- Change `hashFiles('yarn.lock')` to `hashFiles('package-lock.json')` in build-macos.yaml cache key

### Stale index.js cleanup
- Delete `app/mailcore-rs/index.js` — this is the auto-generated napi-rs loader that was superseded by the custom `loader.js` in Phase 1
- Verify `package.json` main field points to `loader.js` (not `index.js`)

### Claude's Discretion
- Exact smoke test script structure and error handling
- Whether to add a CI step name like "Smoke test mailcore-napi" or reuse existing step names
- Any minor workflow YAML cleanup discovered during the edit

</decisions>
