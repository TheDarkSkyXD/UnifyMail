## Decisions Made
- **napi-rs v3 null vs undefined:** `Option<String>` in napi-rs v3 accepts `undefined` from JavaScript but rejects `null`. The field echo test passes `oauth2Token: undefined` instead of `null`. This difference is documented in the established patterns for future reference.
- **Build environment constraint on Windows GNU:** `cargo test` cannot link to an executable due to aws-lc-sys/nanosleep64 MinGW issue. Compile correctness is verified via `cargo check`. Functional verification is achieved via the Electron integration test which loads the freshly-built release `.node` binary. The binary was rebuilt successfully using `napi build --release` with MSYS2 MinGW64 in PATH.
