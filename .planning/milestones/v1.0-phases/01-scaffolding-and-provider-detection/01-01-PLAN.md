    **Step 6: src/provider.rs** (per Research "Complete provider.rs Structure")
    This is the core file. Implement:
    - **Serde structs:** `RawServerEntry`, `RawServers`, `RawProviderEntry` — matching providers.json schema exactly. Use `#[serde(rename = "domain-match")]` etc. Use `#[serde(default)]` on all optional/array fields.
    - **napi export types:** `NetServiceInfo`, `ProviderServers`, `MailProviderInfo` — all with `#[napi(object)]`. Note: napi-rs auto-converts snake_case to camelCase (so `connection_type` becomes `connectionType`, `domain_match` becomes `domainMatch`). This matches the existing TypeScript interface.
    - **Internal Provider struct:** stores `identifier`, `servers` (RawServers), pre-compiled `domain_match_compiled: Vec<(String, regex::Regex)>`, pre-compiled `domain_exclude_compiled: Vec<(String, regex::Regex)>`, and raw `mx_match_patterns: Vec<String>`.
    - **Singleton:** `static PROVIDERS: LazyLock<RwLock<Option<Vec<Provider>>>> = LazyLock::new(|| RwLock::new(None));` — NOT OnceLock (see Research Pitfall 2).
    - **compile_pattern():** wraps pattern with `^...$`, builds case-insensitive regex via `RegexBuilder`. Returns `Option<Regex>`. Invalid patterns return None (skip, not crash).
    - **parse_providers_json():** deserializes `HashMap<String, RawProviderEntry>`, pre-compiles all regex patterns, returns `Vec<Provider>`.
    - **init_from_embedded():** parses embedded JSON, logs count to stderr (`eprintln!`), stores in PROVIDERS.
    - **register_providers():** `#[napi(js_name = "registerProviders")]`. Reads file, parses, MERGES with existing providers (file wins on identifier conflict per CONTEXT.md). Logs merged total.
    - **provider_for_email():** `#[napi(js_name = "providerForEmail")]`. Input validation: throw on empty string or missing '@' (per CONTEXT.md locked decision). Extract domain (lowercase). For each provider: check domain-exclude FIRST (critical for Yahoo), then domain-match. Return `Option<MailProviderInfo>`. Debug logging via `MAILCORE_DEBUG` env var.
    - **Conversion helpers:** `server_to_net_service()` (maps ssl/starttls booleans to "tls"/"starttls"/"clear"), `provider_to_info()`.
