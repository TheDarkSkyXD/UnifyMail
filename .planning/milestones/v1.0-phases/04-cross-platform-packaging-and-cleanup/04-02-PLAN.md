2. **Insert 4 new steps** between "Install Dependencies" and "Lint":

   a. **Setup Rust toolchain:**
   ```yaml
   - name: Setup Rust toolchain
     uses: dtolnay/rust-toolchain@stable
     with:
       targets: x86_64-unknown-linux-gnu
   ```

   b. **Cache cargo:**
   ```yaml
   - name: Cache cargo registry and build
     uses: actions/cache@v4
     with:
       path: |
         ~/.cargo/registry/index/
         ~/.cargo/registry/cache/
         ~/.cargo/git/db/
         app/mailcore-rs/target/
       key: linux-x64-cargo-${{ hashFiles('app/mailcore-rs/Cargo.lock') }}
       restore-keys: |
         linux-x64-cargo-
   ```

   c. **Build napi-rs addon:**
   ```yaml
   - name: Build napi-rs addon (Linux x64)
     working-directory: app/mailcore-rs
     run: npx @napi-rs/cli build --release --target x86_64-unknown-linux-gnu
   ```

   d. **Binary size gate (SCAF-04 — Linux x64 ONLY):**
   ```yaml
   - name: Check binary size (8MB gate)
     working-directory: app/mailcore-rs
     run: |
       BINARY=$(ls *.node | head -1)
       SIZE_KB=$(du -k "$BINARY" | cut -f1)
       echo "Binary: $BINARY -- size: ${SIZE_KB}KB"
       if [ "$SIZE_KB" -gt 8192 ]; then
         echo "FAIL: ${SIZE_KB}KB exceeds 8192KB (8MB) limit"
         echo "Investigate with: cargo install cargo-bloat && cargo bloat --release -n 10"
         exit 1
       fi
       echo "PASS: Within 8MB limit"
   ```

3. **Insert smoke test step** between the napi build steps and "Lint":
   ```yaml
   - name: Smoke test -- verify mailcore-napi loads
     working-directory: app
     run: |
       node -e "
         const m = require('mailcore-napi');
         if (typeof m.providerForEmail !== 'function') { console.error('FAIL: providerForEmail not a function'); process.exit(1); }
         const r = m.providerForEmail('test@gmail.com');
         if (!r || typeof r !== 'object') { console.error('FAIL: no result'); process.exit(1); }
         console.log('PASS: mailcore-napi loaded, provider:', r.displayName || JSON.stringify(r));
       "
   ```

   ```yaml
   - name: Setup MSYS2 (provides dlltool.exe for GNU target)
     uses: msys2/setup-msys2@v2
     with:
       msystem: MINGW64
       install: mingw-w64-x86_64-toolchain
   ```

   b. **Generate libnode.dll import library:**
   ```yaml
   - name: Generate libnode.dll import library
     shell: msys2 {0}
     run: |
       NODE_EXE=$(which node)
       gendef "$NODE_EXE"
       dlltool --dllname node.exe --def node.def --output-lib /tmp/libnode.dll
       echo "LIBNODE_PATH=$(cygpath -w /tmp)" >> $GITHUB_ENV
   ```

   c. **Setup Rust toolchain:**
   ```yaml
   - name: Setup Rust toolchain (Windows GNU)
     uses: dtolnay/rust-toolchain@stable
     with:
       targets: x86_64-pc-windows-gnu
   ```

   d. **Cache cargo:**
   ```yaml
   - name: Cache cargo registry and build
     uses: actions/cache@v4
     with:
       path: |
         ~/.cargo/registry/index/
         ~/.cargo/registry/cache/
         ~/.cargo/git/db/
         app/mailcore-rs/target/
       key: win-x64-gnu-cargo-${{ hashFiles('app/mailcore-rs/Cargo.lock') }}
       restore-keys: |
         win-x64-gnu-cargo-
   ```

   e. **Build napi-rs addon:**
   ```yaml
   - name: Build napi-rs addon (Windows x64 GNU)
     shell: msys2 {0}
     working-directory: app/mailcore-rs
     env:
       LIBNODE_PATH: ${{ env.LIBNODE_PATH }}
     run: |
       export PATH="/mingw64/bin:$PATH"
       npx @napi-rs/cli build --release --target x86_64-pc-windows-gnu
   ```

3. **Insert smoke test step** after the napi build and before "Lint":

   (This step uses the default shell, NOT msys2 — Node.js is the setup-node version.)

**CRITICAL notes:**
- Use `x86_64-pc-windows-gnu` target, NOT `x86_64-pc-windows-msvc` — the project uses GNU toolchain (established Phase 1 decision).
- The `msys2 {0}` shell is required for gendef, dlltool, and the napi build step because they need MinGW64 tools in PATH.
