Create `app/mailcore-rs/src/imap.rs` with the full `testIMAPConnection` implementation. Follow the architecture from 02-RESEARCH.md patterns 1-9.

**Exported napi types:**

1. `IMAPConnectionOptions` (#[napi(object)]):
   - hostname: String
   - port: u32
   - connection_type: Option<String>  (JS: connectionType -- "tls" | "starttls" | "clear")
   - username: Option<String>
   - password: Option<String>
   - oauth2_token: Option<String>  (JS: oauth2Token)

2. `IMAPConnectionResult` (#[napi(object)]):
   - success: bool
   - error: Option<String>
   - error_type: Option<String>  (JS: errorType -- per user decision: "connection_refused" | "timeout" | "tls_error" | "auth_failed" | "unknown")
   - capabilities: Option<Vec<String>>

**Internal structure:**

1. `XOAuth2` struct implementing `async_imap::Authenticator`:
   - process() returns `format!("user={}\x01auth=Bearer {}\x01\x01", self.user, self.access_token)`
   - CRITICAL: Use `\x01` (SOH byte), NOT `\x00` (null)

2. `make_tls_config()` -> `ClientConfig`:
   - Use `ClientConfig::with_platform_verifier()` from `rustls_platform_verifier::ConfigVerifierExt`
   - Strict validation only (no allowInsecureSsl per user decision)

3. `make_server_name(host: &str)` -> `ServerName<'static>`:
   - Parse as IpAddr first; if IP, use `ServerName::IpAddress(ip.into())`
   - Otherwise use `ServerName::try_from(host.to_string())`

4. `extract_capabilities(caps: &Capabilities)` -> `Vec<String>`:
   - Map: IDLE->idle, CONDSTORE->condstore, QRESYNC->qresync, COMPRESS=DEFLATE->compress, NAMESPACE->namespace, AUTH=XOAUTH2->xoauth2, X-GM-EXT-1->gmail

5. `classify_error(err: &(dyn std::error::Error))` -> `String`:
   - Check error chain for:
     - "Connection refused" / io::ErrorKind::ConnectionRefused -> "connection_refused"
     - rustls/TLS errors -> "tls_error"
     - "authentication" / "login" / IMAP NO response -> "auth_failed"
     - Default -> "unknown"
   - Include hostname:port in error message per user decision

6. `auth_and_capabilities<S>(client, username, password, oauth2_token)` -> `Result<IMAPConnectionResult>`:
   - Generic over S: AsyncRead + AsyncWrite + Unpin + Send + Debug
   - If oauth2_token present: `client.authenticate("XOAUTH2", XOAuth2{...}).await.map_err(|(err, _)| err)?`
   - Else if password present: `client.login(username, pass).await.map_err(|(err, _)| err)?`
   - Else: return error "No password or OAuth2 token provided"
   - Call `session.capabilities().await?`
   - Call `session.logout().await` (ignore error)
   - Return success result with capabilities

7. `connect_tls(host, port)` -> `Result<Client<TlsStream<TcpStream>>>`:
   - TcpStream::connect -> TlsConnector::connect -> Client::new

8. `connect_starttls(host, port)` -> `Result<Client<TlsStream<TcpStream>>>`:
   - TcpStream::connect -> Client::new (reads greeting) -> run_command_and_check_ok("STARTTLS") -> into_inner() -> TlsConnector::connect -> Client::new
   - WARNING: The second Client::new after STARTTLS may hang waiting for a greeting that doesn't exist. If this happens during testing, use Connection::new or handle it at a lower level. See research Pitfall 1. The mock tests in Plan 02 will validate this.

9. `connect_clear(host, port)` -> `Result<Client<TcpStream>>`:
   - TcpStream::connect -> Client::new

10. `do_test_imap(opts)` -> `Result<IMAPConnectionResult>`:
    - Match connection_type: "tls" | "starttls" | "clear" (default: "tls")
    - Connect, then auth_and_capabilities
    - Debug log (MAILCORE_DEBUG=1): "testIMAPConnection -> Rust: {host}:{port} ({conn_type})"

11. `test_imap_connection(opts)` (#[napi(js_name = "testIMAPConnection")]):
    - `tokio::time::timeout(Duration::from_secs(15), do_test_imap(&opts)).await`
    - Ok(Ok(result)) -> Ok(result)
    - Ok(Err(e)) -> Ok(IMAPConnectionResult { success: false, error: formatted with host:port, error_type: classify_error, capabilities: None })
    - Err(_elapsed) -> Ok(IMAPConnectionResult { success: false, error: "Connection to {host}:{port} timed out after 15 seconds", error_type: Some("timeout"), capabilities: None })
    - CRITICAL: Use `#[napi(js_name = "testIMAPConnection")]` -- without this, napi auto-converts to testImapConnection which breaks callers

**Error type for internal errors:**
Use `Box<dyn std::error::Error + Send + Sync>` as the internal error type. Only convert to napi::Error at the top-level napi function boundary (and only for truly unexpected errors, not connection failures).

**Debug logging pattern (from Phase 1):**
```rust
if std::env::var("MAILCORE_DEBUG").as_deref() == Ok("1") {
    eprintln!("testIMAPConnection -> Rust: {}:{} ({})", host, port, conn_type);
}
```
