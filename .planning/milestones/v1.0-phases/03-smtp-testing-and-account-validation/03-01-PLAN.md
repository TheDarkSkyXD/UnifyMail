**smtp.rs implementation (mirrors imap.rs structure):**

1. Define napi-exported types:
   - `SMTPConnectionOptions` (#[napi(object)]): hostname (String), port (u32), connection_type (Option String -- "tls"|"starttls"|"clear"), username (Option String), password (Option String), oauth2_token (Option String)
   - `SMTPConnectionResult` (#[napi(object)]): success (bool), error (Option String), error_type (Option String)
   - Use snake_case fields; napi-rs auto-converts to camelCase in JS.

2. Implement `pub async fn do_test_smtp(opts: &SMTPConnectionOptions) -> InternalResult<SMTPConnectionResult>`:
   - Determine connection mode from `opts.connection_type` (default "tls"):
     - "tls": `AsyncSmtpTransport::<Tokio1Executor>::relay(hostname)?.port(port).build()`
     - "starttls": `AsyncSmtpTransport::<Tokio1Executor>::starttls_relay(hostname)?.port(port).build()`
     - "clear": `AsyncSmtpTransport::<Tokio1Executor>::builder_dangerous(hostname).port(port).build()`
   - If credentials provided (username + password OR username + oauth2_token):
     - Password: `Credentials::new(user, pass)` + `.authentication(vec![Mechanism::Login])`
     - XOAUTH2: `Credentials::new(user, token)` + `.authentication(vec![Mechanism::Xoauth2])` -- lettre handles XOAUTH2 SASL encoding internally; no custom Authenticator needed (unlike Phase 2 async-imap)
   - If no credentials: connect-only mode (EHLO + NOOP, no AUTH)
   - Call `transport.test_connection().await` -- this does EHLO -> AUTH (if creds) -> NOOP
   - On Ok(true): return SMTPConnectionResult { success: true, error: None, error_type: None }
   - On Err(e): classify error and return SMTPConnectionResult { success: false, error: Some(msg), error_type: Some(classify_smtp_error(&e)) }
   - Debug logging with MAILCORE_DEBUG=1 env var (same as Phase 1+2 pattern)

3. Implement `classify_smtp_error(err: &lettre::transport::smtp::Error) -> String`:
   - Same categories as IMAP: "connection_refused", "tls_error", "auth_failed", "unknown"
   - "timeout" is set separately by the outer timeout handler
   - Check error Display string for known patterns (same approach as imap.rs classify_error)

4. Implement napi export `#[napi(js_name = "testSMTPConnection")] pub async fn test_smtp_connection(opts: SMTPConnectionOptions) -> napi::Result<SMTPConnectionResult>`:
   - Wrap `do_test_smtp(&opts)` in `tokio::time::timeout(Duration::from_secs(15), ...)`
   - On timeout: return SMTPConnectionResult { success: false, error: timeout message, error_type: "timeout" }
   - On inner error: return classified error result
   - Promise ALWAYS resolves (never rejects) -- same pattern as testIMAPConnection

5. Use `type BoxError = Box<dyn std::error::Error + Send + Sync>;` and `type InternalResult<T> = std::result::Result<T, BoxError>;` (duplicated from imap.rs, not imported -- each module is self-contained per Phase 2 pattern)

**smtp_tests.rs implementation (mirrors imap_tests.rs pattern):**

1. Create mock SMTP server using `TcpListener::bind("127.0.0.1:0")` for random port per test (no shared state, no TEST_MUTEX).

2. Mock SMTP server protocol handler:
   - Send `220 mock.smtp.test ESMTP ready\r\n` greeting on connect
   - Handle `EHLO`: respond `250-mock.smtp.test\r\n250-AUTH LOGIN XOAUTH2\r\n250 OK\r\n`
   - Handle `AUTH LOGIN`: multi-step base64 dialog -- `334 VXNlcm5hbWU6\r\n` (Username:), read base64 user, `334 UGFzc3dvcmQ6\r\n` (Password:), read base64 pass, respond `235 2.7.0 Authentication successful\r\n` (or `535 5.7.8 Auth failed\r\n` for reject mode)
   - Handle `AUTH XOAUTH2`: read base64 token, respond `235 2.7.0 OK\r\n` (or `535` for reject)
   - Handle `NOOP`: respond `250 OK\r\n`
   - Handle `QUIT`: respond `221 Bye\r\n`
   - Use a `MockSmtpMode` enum: `AcceptAll`, `RejectAuth`, `NeverRespond` (for timeout test)

3. Tests call `do_test_smtp()` directly (not the napi function) with `tokio::time::timeout(Duration::from_secs(3), ...)` for fast test execution (same pattern as imap_tests.rs).

4. Include a `classify_error_str` helper in the test file that mirrors smtp.rs classify logic for assertion purposes (same pattern as imap_tests.rs).

IMPORTANT ANTI-PATTERNS TO AVOID:
- Do NOT use lettre default features (OpenSSL conflict)
- Do NOT forget `.authentication(vec![Mechanism::Xoauth2])` when using XOAUTH2 credentials
- Do NOT manually read SMTP 220 greeting -- lettre handles this internally (unlike async-imap in Phase 2)
- lettre `.timeout()` is per-command, NOT total -- always use outer tokio::time::timeout
- `#[napi(js_name = "testSMTPConnection")]` must be all-caps SMTP (napi-rs auto-converts to testSmtpConnection which is wrong)
